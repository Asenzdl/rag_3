## Task 1.5 Prompt 模板工程与多语言支持

### 任务目标
设计支持中文提问、英文文档、中文回答的 Prompt 模板，确保生成质量和引用规范。

### 涉及文件
- `src/generation/prompts.py`

### 面试级知识点
- **System Message vs Human Message** 的角色分工：System 设定全局行为（语言、引用格式），Human 提供具体问题和上下文。
- **跨语言 RAG 的 Prompt 设计**：明确指示"基于英文文档回答，但用中文输出"，防止模型混淆语言。
- **Few-shot 示例的作用**：在 Prompt 中加入 1-2 个示例，可显著提升引用格式的遵从度。

### 生产级注意事项
- **模板变量标准化**：使用 `ChatPromptTemplate` 的 `MessagesPlaceholder` 管理动态上下文，变量命名遵循 `context`、`question`、`chat_history` 约定。
- **Prompt 版本管理**：在 `prompts.py` 中以字典形式存储不同版本（如 `V1`、`V2`），便于 A/B 测试。
- **长度控制**：在 Prompt 中明确指示"如果上下文不足，请如实回答'我不知道'"，防止幻觉。

### 验收标准
- 在临时脚本中，手动填入一段英文文档片段和中文问题，调用 LLM 验证输出语言和引用格式。
- 输出中必须包含引用标记（如 `[1]`），并在末尾列出来源 URL。
- 当上下文不包含答案时，模型应回答"根据现有文档，我无法回答该问题"。
