## Task 5.6 部署文档与演示准备

### 任务目标
编写完整的部署文档和用户指南,准备一份可用于面试演示的"电梯演讲"脚本和 Demo 视频素材,确保项目可清晰展示。

### 涉及文件
- `README.md`
- `DEPLOYMENT.md`
- `docs/demo_script.md`

### 面试级知识点
- **技术文档的写作规范**:清晰的结构(简介、快速开始、配置、API 参考、常见问题)、代码块高亮、环境变量说明。好的文档是开源项目吸引贡献者的关键。
- **README 驱动的开发**:在动手写代码前先写 README,明确项目目标和使用方式,是业界推崇的实践。
- **Demo 演示技巧**:准备 3-5 个代表性场景(本地知识库问答、多轮追问、网络搜索 fallback),展示系统的核心价值,而非罗列所有功能。

### 生产级注意事项
- **文档的持续更新**:每次功能变更必须同步更新文档。建议在 CI 中检查文档链接的有效性(使用 `markdown-link-check`)。
- **提供一键启动脚本**:编写 `scripts/start.sh` 封装 `docker compose up` 和初始化检查,降低用户使用门槛。
- **演示环境与生产环境分离**:演示时可使用较小的模型(如 `gpt-4o-mini`)以降低成本,通过环境变量切换。
- **录制演示视频**:使用 Loom 或 OBS 录制 3 分钟内的演示视频,嵌入到 README 或项目主页,便于面试官快速了解成果。

### 验收标准
- `README.md` 包含:项目简介、功能特性、技术栈、快速开始(Docker 方式)、本地开发指南、API 示例。
- `DEPLOYMENT.md` 包含:服务器部署步骤、环境变量清单、监控配置建议、备份策略。
- `docs/demo_script.md` 包含 5 个演示场景的对话脚本和预期输出,面试时可照此演示。
- 在 Docker 环境外,运行 `./scripts/start.sh` 能自动检查依赖并启动服务。
- 录制一段 2 分钟的视频,展示从提问到获得带引用回答的完整流程,并在 `README` 中提供链接。
