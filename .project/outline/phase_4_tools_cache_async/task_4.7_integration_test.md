## Task 4.7 Phase 4 端到端集成测试与文档更新

### 任务目标
完成 Phase 4 所有新增功能的集成测试,确保工具调用、缓存、异步处理协同工作无冲突,并更新项目文档以反映当前系统能力。

### 涉及文件
- `tests/test_e2e_phase4.py`
- `AGENTS.md`
- `README.md`(若存在)

### 面试级知识点
- **集成测试的策略**:模拟真实用户场景的多轮对话,覆盖:① 本地知识库命中;② 触发网络搜索;③ 精确缓存命中;④ 语义缓存命中;⑤ 异步并发请求。
- **Mock 外部依赖**:在测试环境中,Tavily API 和 LLM API 应使用 Mock 对象,避免消耗真实配额和成本。使用 `unittest.mock` 或 `pytest-mock`。
- **测试数据管理**:为缓存测试准备固定的 QA pairs,确保每次运行结果可重现。

### 生产级注意事项
- **测试隔离**:每个测试用例使用独立的 `thread_id` 和临时缓存数据库,避免测试间相互干扰。
- **环境变量切换**:测试环境通过 `.env.test` 文件配置(如 `TAVILY_API_KEY=mock`、`ENABLE_CACHE=true`),与开发/生产环境隔离。
- **持续集成准备**:确保所有测试可通过 `pytest` 一键运行,为后续 CI/CD 流水线做准备。
- **文档更新**:在 `AGENTS.md` 中新增 Phase 4 章节,描述新增功能(网络搜索、缓存、异步)、配置方式、性能基准数据。面试时可直接引用此文档作为系统设计说明。

### 验收标准
- 编写 `tests/test_e2e_phase4.py`,包含至少 5 个集成测试用例:
  1. 本地知识库问答成功
  2. 触发 Tavily 网络搜索并返回结果
  3. 精确缓存命中(第二次相同问题返回缓存)
  4. 语义缓存命中(变体问题命中缓存)
  5. 异步并发 10 请求全部成功
- 所有测试通过,且运行时间 < 30 秒(不含网络延迟,因使用 Mock)。
- 更新 `AGENTS.md`,记录 Phase 4 新增模块、配置参数、性能基准(如缓存命中率 40%、P95 延迟降低 30%)。
- 生成一份 Phase 4 完成总结报告(`docs/phase4_summary.md`),供面试时展示。
